Termes utilisés

  • GIT : logiciel de versionning, c'est aussi une commande dans la console : git [nom de l'option]
  • INIT (cmd) : "INITER", c'est faire devenir un dossier en local sur votre disque dur un depôt, un dossier ou Git est en fonctionnement (initialisé). Le DOSSIER LOCAL devient un DEPOT
  • TRACKER (v.) : ajouter un fichier à la liste de suivi.
  • COMMITER (v.) : sauvegarder via un commit, le commit est l'action d'efffectuer une sauvegarde d'un projet
  • BACKUP (objet) : une sauvegarde issue d'un commit.
  • DEPOT (objet) : dossier ou on a initialisé Git : nommé aussi repository
  • REMOTE (objet) : une machine. Cela peut être un serveur comme votre pc, un télephone portable...
    origin est le nom du remote du serveur Github en ligne
  • UNTRACKED (message) : fichier qui n'a pas été ajouté à l'index, ou nommé stagging area.
  • INDEX ou aussi appelé STAGGING AREA (objet) : zone tampon dans laquelle sont ajoutées les fichiers locaux, ce sont ces fichiers qui au prochain commit seront enregistrés dans un BACKUP
  • BRANCHE : le voir comme une voie parralèle au projet principal ou on développe avec une stagging area distincte et des backups distints de la branche principale du projet. Cela sert a developper des Fonctionnalites, qui une fois opérationnelles, sont fusionnés avec la branche principale, appelée master.

Récupérer un projet existant sur Github.com et le retravailler (un contributeur)

  1. git clone [projet]
  2. cd [projet]
  3. ..travailler..
  4. git status
  5. git add
  6. git status
  7. git commit -m "nom du commit"
  8. git push -u origin master

Créer son dépôt à partir d'un dossier local et versionner depuis ce dossier

  1. mkdir [nom_dossier] -> créer son dossier local
  2. cd [nom_dossier] -> se placer dans le dossier crée
  3. git init -> intialiser le projet, le transformer en repository (dépôt)
  4. git commit -m "[message pour initialiser son commit]" -> réaliser un commit, on obtient ainsi un backup
  5. git add . -> ajouter tous les fichiers à son index (avec le .), sinon changer le . par les fichiers de votre choix
  6. git log -> affiche la liste de tous les commits réalisés. Il sont rangés par ordre chronologique descendant. un commit contient: un SHA (identifiant unique), un auteur, une date, une description (celle que vous avez indiqué)
  7. rien a valider, la copie de travail est propre -> message signifiant que les fichiers suivis n'ont pas été modifiés depuis le dernier commit.
  8. Il faut d'abord suivre les fichiers pour ensuite commit. Commit n'est pas pusher.

Se postionner sur un commit donné, revenir en arrière

  1. git checkout [SHA_du_commit] -> se repositionner sur un commit spécifique (remonter dans le temps)
  2. git checkout master -> revenir à la branche principale (= le commit le plus récent)
  3. git reset --hard [SHA_du_commit] -> annuler tous les changements non encore commités. Cela dit : reviens a ce commit la avec l'état des fichiers de ce commit la et annule tous les commit qui ont été fait après
  4. git reset --soft [SHA_du_commit] -> on dit a git : reviens a ce commit la, mais n'anule pas toutes les modifications que j'ai fait jusque là
  5. git commit --amend --no-edit -> enregistre mes modifcations a mon commit actuel dans lequel je suis placé, mais sanc créer un nouveau commit

Remote externe : versionner sur un Dépôt GitHub distant en ligne (le serveur gitHub)

gitHub est le service en ligne. Créer un compte et suivre les instructions. Ne pas confondre git le logiciel de versionning à installer sur sa machine de GitHub le service un ligne ou on peut gérer des dépots distants.

Récupérer un projet depuis le GitHub sur sa machine locale

Aller sur GitHub, se placer sur le projet, puis copier la web url

  1. git clone [web_url] -> copie en local le projet sur la machine
  2. git init -> le transforme en dépôt sur la machine existante

Créer un dépot distant géré sur con compte GitHub et le gérer localement

  1. Suivre les captures ci-dessus et donner un nom au projet, une description et un statut ( l'option INITIALISE with a README clone le repository crée sur la machine en direct)
  2. Récupérer la web url du projet
  3. Se positionner dans le dossier local de son choix
  4. git clone -> le projet est récupéré
  5. git init

Créer un dépot distant géré sur con compte GitHubpuis envoyer une PROJET DEJA EXISTANT localement vers le repository de Github (pushing)

  1. Même procédure que ci-dessus suivre les capture d'ecran, puis après que le Reposotory distant soit crée
  2. Se placer dans le dossier local qui est DEJA init comme Repository local
  3. git remote add origin [web_url_du_projet] -> On dis a son repository qu'on lui ajoute le nom du remote du serveur Github en ligne avec le lien du projet
  4. git push -u origin master -> crée un backup distant sur le repository distant

Faire un push à chaque fois qu'on veut crée un backup distant. Dans le projet GitHub clquer sur les liens commit pour voir les différents commit.

Récupérer les données du dépôt en ligne (si on travail à plusieurs ou si on modifie un fichier en ligne) régulièrement pour avoir la dernière version

Rappel : un remote est une machine qui heberge. Origin = le dépôt GitHub en ligne, le serveur.

  1. Se place dans son dossier local préalablement inité
  2. git pull origin master -> on "pull", on tire les données du Repository distant vers son dossier local

Utiliser le système de branches

  1. git branch -> retourne les branches présentes, et ajoute une étoile sur la branche dans laquelle on est placé
  2. git branch [nouvelle_branche] -> créer une nouvelle branche
  3. git checkout [nom_de_branche] -> se positionner dans la branche nouvellement crée
  4. Toujours commiter avant de changer de branche

Mise à jour des branches et fusion avec la branche master

  1. git checkout master -> se placer dans la branche principale
  2. git merge [nom_deLa_branche_declinee] -> fusionner la branche avec la branche principale

Savoir qui a fait une modification

  1. git blame [nomFichierExtensionIncluse] -> liste toutes les modifications effectuées sur un fichier ligne par ligne
  2. git show [debut_du_SHA] -> renvoie directement les détails du commit recherché

Ignorer des fichiers (fondamental) : il est impératif d'ignorer les fichiers de configuration contenant les mots de passe en cas de push

  1. Se placer dans son repository
  2. touch .gitignore -> crée un fichier de configuration
  3. ouvrir le fichier .gitignore
  4. rajouter ligne par ligne en mettant le chemin absolu du REPOSITORY les fichiers à ignorer et sauvegarder le fichier
  5. git add -A

Mettre de coté les modifications avant d'une branche avant de passer dans une autre

  1. git stash -> met de coté les modifications en cours
  2. se rendre dans une autre, branche, faire ses commit, modifier, etc...
  3. revenir dans la branche ou on travaillait
  4. git stash pop -> récuperer les modifications qui avaient été mis de coté et retravailler

En résumé

Intialiser un projet

cd [dossier_du_projet] > git init

Réaliser un commit

git add . > git commit -m "nomCommit"

Ajouter un remote

git remote add origin [lien_http_projet]

Réaliser un push : LOCAL >>> REMOTE

git push -u origin master

Réaliser un pull : REMOTE >>> LOCAL

Sur un nouveau dossier -> d'abord se positionner dans le dossier -> initialiser avec git init -> ajouter le remote [lien_http_projet] -> puis faire un pull

ou

Sur un nouveau dossier -> d'abord se positionner dans le dossier -> initialiser avec git init -> git clone [lien_http_projet]

git pull origin master

Supprimer un remote

git remote rm origin > git remote rm upstream

Désinitialiser un projet

rm -rf .git

Connaitre son dépôt git actuel

git remote show origin
